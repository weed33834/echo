# 安全策略

## 报告漏洞

如果你发现了安全漏洞，请**不要**在公开 Issue 中提交。

请发送邮件至：security@example.com（请替换为实际邮箱）

报告中请包含：
- 漏洞描述
- 复现步骤
- 影响范围
- 建议的修复方案（如有）

我们会在收到报告后 48 小时内回复，并在修复后公开致谢。

## 已知限制

### 开发服务器

Vite 5.x 使用的 esbuild 开发服务器存在 [GHSA-67mh-4wv8-2f99](https://github.com/advisories/GHSA-67mh-4wv8-2f99) 漏洞，可能导致开发服务器被任意网站跨域访问。

**影响范围**：仅影响本地开发环境，不影响生产构建。

**建议**：
- 不要在公共网络中运行开发服务器
- 生产环境使用 `npm run build` + `npm run preview` 或部署到静态服务器

### API Key 存储

本项目的 API Key 存储在浏览器 `localStorage` 中，这是纯前端应用的常见做法。

**注意**：
- 不要在公共电脑上保存 API Key
- 分享浏览器配置文件可能导致 Key 泄露
- 建议使用各 AI 服务商的用量限制功能

### 沙箱机制

项目内置安全沙箱（`src/services/sandbox.js`），对 AI 工具调用进行：
- 参数校验（类型、范围、长度）
- 超时控制（防止无限循环）
- 注入检测（Prompt Injection 防护）
- 输出验证（结果格式校验）

但沙箱不能替代服务器端验证。如果将本项目改造为有后端的应用，请在服务端重新实现所有校验逻辑。

## 安全护栏

AI 对话系统内置五类安全护栏：

| 类型 | 触发条件 | 处理方式 |
|------|----------|----------|
| 医疗护栏 | 涉及疾病、用药 | 仅从五行养生角度分析，建议就医 |
| 法律护栏 | 涉及纠纷、官司 | 不预测输赢，建议咨询律师 |
| 财务护栏 | 涉及投资、理财 | 不做收益保证，强调风险 |
| 危机护栏 | 检测到自残倾向 | 优先关注安全，提供援助热线 |
| 绝对化护栏 | 要求确定预测 | 使用软性表述，说明局限性 |
